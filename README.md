# The Ball

The Ball is a 3D action survival game made in Unity.

The player avoids enemies, collects stars, and survives enemy waves. The main focus of the project was to create a simple and clear user interface that makes the game easy to understand and play.

## Project Overview

For this project, I focused on user centred design. I wanted the game to feel clear, simple, and easy to use. The player needs to keep track of health, stars, and wave progress, so the interface was built around showing that information in a straightforward way.

## Main Idea

The main idea was to make a game UI that supports the player without getting in the way. I kept the design simple so players could focus on the game. I also thought about accessibility and how to make the game feel more welcoming for different players.

## Main Features

- Main menu
- Lobby screen
- Gameplay HUD
- Pause menu
- Health icons
- Star counter
- Wave counter
- Countdown timer
- Win screen
- Game over screen
- Audio feedback
- Basic accessibility features

## What I Worked On

For this project, I worked on the menu design, UI layout, mock ups, gameplay HUD, pause menu, accessibility ideas, and the scripts that connect the UI to the game. I also worked on player feedback such as sounds, panel transitions, and simple animations.

## UI Design

I kept the interface simple and easy to read. I used heart icons for health and stars for progress so the player can understand things quickly. I also made the countdown timer large and easy to see so players know when the next wave is about to start.

## Prototype

The prototype was built in Unity using C#. It includes a main menu, a lobby, gameplay UI, and a pause menu. I also used scripts to manage button actions, panel changes, player movement, star collection, and extra UI features such as camera movement for the scoreboard and spinning stars.

## Accessibility

Accessibility was an important part of this project. I added icon based UI, a flashing countdown before waves, large buttons, and keyboard support through the pause menu. I wanted the game to feel easier to use for players with different needs and different levels of experience.

## Tools Used

- Unity

## What I Learned

This project helped me improve my understanding of UI design, player feedback, accessibility, menu systems, and scripting in Unity. It also helped me see how much small UI details matter when it comes to the player experience.

## Challenges

One of the main problems was getting the volume sliders to work properly. I also had issues with UI anchors moving on different screen sizes. Some features, such as the scoreboard and kill feed, were planned but not fully finished. These problems showed me that testing and planning are really important.

## Future Improvements

If I continued this project, I would improve the overall look of the UI, add a working scoreboard, create a kill feed, improve the controls screen, and spend more time testing across different screen sizes. I would also like to make the whole design feel less basic and more polished.

## Summary

The Ball was a good project for learning about UI design in games. It helped me build my skills in Unity, scripting, accessibility, and player centred design. It also showed me how important it is to keep things clear, simple, and useful for the player.
